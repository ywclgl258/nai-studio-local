# NAI Studio Desktop · 用户使用手册

> Tauri + Rust 重构版的 NAI Studio — 单文件 .exe, 体积 8MB, 启动 <1s, 内存 44MB
>
> 适用版本: **v2.0.0** 及以上

---

## 📦 1. 安装

### 系统要求
- **Windows 10/11** (自带 WebView2 Runtime,无需额外安装)
- 无需 XAMPP / PHP / Node.js / MySQL

### 部署
1. 把 `nai-studio-desktop.exe` (8MB) 拷到任意位置(桌面、U 盘、Program Files 都行)
2. 双击运行
3. 首次启动会弹窗口,稍等 1 秒左右 WebView 加载完即可使用

### 数据位置
所有数据都存在 Windows 用户目录下,**卸载 EXE 也不会丢数据**:

```
%APPDATA%\nai-studio-desktop\              ← Win + R 输入 %APPDATA% 回车
├── nai-studio.db                         ← SQLite 数据库
├── data/                                 ← 业务数据
├── data-tpl/                             ← 数据模板
├── logs/                                 ← 运行日志(出问题先看这里)
└── storage/                              ← 图片 + 缩略图 + 上传 + upscaler
    ├── images/                           ← 生成的大图
    ├── thumbs/                           ← 缩略图
    ├── upscales/                         ← Real-ESRGAN 放大后的图
    ├── uploads/                          ← 上传 / 导入的图
    ├── cache/                            ← 临时缓存(可定期清理)
    ├── tag-previews/                     ← 标签预览图
    └── tools/realesrgan/                 ← Real-ESRGAN 程序和模型
```

## ⚙️ 2. 首次配置

### 2.1 NAI API Key (必需)
1. 打开 **设置 → API Key**
2. 点 **"+ 新增 Key"**
3. 输入 `label`(随便起名,如"主号""备用")
4. 粘贴 NAI 官网的 API key (Settings → Account → API Key)
5. 点 **保存**

> 💡 支持**多 Key 轮换** — 一个被限流 / 失败会自动试下一个。  
> 💡 Key 用 **AES-256-GCM 加密** 存到本地,跟旧版 PHP 项目格式一致。

### 2.2 AI 助手 (可选)
用于 AI 拆 prompt / 分析图片,需要单独配:

1. 打开 **设置 → AI 助手**
2. 选 Provider(DeepSeek / OpenAI / SiliconFlow / Ollama)
3. 填 base URL + API key + 模型
4. 点 **测试连接** 验证

### 2.3 代理 (可选)
国内直连 NAI 经常被 Cloudflare WAF 拦,需要代理:

1. 打开 **设置 → 代理**
2. 勾选"启用代理"
3. 填 `http://127.0.0.1:7890`(Clash 默认端口)或 `http://127.0.0.1:10809`(v2rayN)
4. 点 **测试** 验证能否走通

## 🎨 3. 出图 (核心功能)

### 3.1 单张生图
1. 在左侧主输入框写 **prompt**(英文逗号分隔的 tag 串),例:
   ```
   masterpiece, best_quality, 1girl, solo, hatsune_miku, long_hair, twintails, blue_eyes, smile, school_uniform, standing, cherry_blossoms, outdoors
   ```
2. 选 **model**(V3 / V4 / V4.5-Curated / V4.5-Full)
3. 选 **sampler / steps / scale / size**
4. 点 **生成** 按钮
5. 右侧预览区显示进度,几秒到几十秒出图

### 3.2 角色 / 姿势提示词
在 prompt tab 切换:
- **角色 tab** — 写角色描述(会拼到主 prompt 后)
- **姿势 tab** — 写姿势描述(会拼到主 prompt 后)

最终 prompt = `主 prompt + 角色 + 姿势`(逗号连接)

> 💡 角色和姿势可以**保存为预设**,下次一键载入。  
> 💡 预设库在侧边栏可展开浏览。

### 3.3 批量生图
1. 点主界面 **"批量"** 按钮
2. 选 N 张参考图(本地)
3. 配置每张用的 prompt 模板
4. 启动队列,后台依次跑

> 批量会按顺序处理,失败自动重试一次。进度在底部历史条显示。

### 3.4 Vibe Transfer (风格迁移)
上传 1 张参考图,AI 学参考图的风格生成新图:
1. 切到 **Vibe** 标签
2. 拖入参考图
3. 调信息强度(0-1)
4. 写 prompt + 生成

### 3.5 Precise / Mask Editor (局部重绘)
1. 上传原图
2. 在 mask editor 里涂需要重绘的区域
3. 写局部 prompt + 生成

## 🖼️ 4. 画廊 / 历史

### 4.1 浏览
启动后默认显示**画廊**视图(底部历史条可切换):
- 缩略图列表,带 prompt / seed / 模型信息
- 鼠标悬停看大图
- 点击进入详情

### 4.2 操作
右键缩略图有菜单:
- **收藏 ★** — 加到收藏夹(清理时不会被删)
- **Upscale 2x / 4x / 8x** — 调 Real-ESRGAN 放大
- **用此图作为 Vibe / Precise / Mask 输入**
- **复制 prompt / seed / 设置**
- **下载 / 删除**

### 4.3 打包下载
1. 选多张(勾选框)
2. 点 **打包 ZIP** 按钮
3. 自动下载 `nai-studio-YYYYMMDD.zip`

## 🔍 5. 标签 / 画师

### 5.1 Tag 库
- **本地 tag** — 你手动加的 tag(带中文翻译)
- **Danbooru 在线搜索** — 输入英文 tag,自动翻译成中文
- **分类浏览** — 按 general / artist / character / meta 过滤

操作:
- 搜索 → 点 tag → 自动加到主 prompt 输入框
- 长 tag 可点 **星标** 收藏

### 5.2 画师库
- 本地保存的画师预设(artist:ciloranko 等)
- 点 **搜索 Danbooru** 在线拉新画师(post_count / example image)
- 可批量抓取画师示例图

## 🧹 6. 清理 / 维护

### 6.1 清理孤儿文件
数据库里没引用但 storage 目录里还在的图(导入或迁移残留):
- 打开 **设置 → 清理**
- 选 level: `orphans` / `cache` / `logs` / `all`
- 点清理,会扫 storage + 清掉引用不到的图

### 6.2 备份 / 迁移
整个 `nai-studio-desktop/` 文件夹 = 你的全部数据,直接 zip 备份即可:
```
zip -r backup.zip nai-studio-desktop/
```
换电脑时解压到新机器的 `%APPDATA%\nai-studio-desktop\` 即可。

## ❓ 7. 常见问题

### Q1: 启动后窗口空白 / 不显示
- **WebView2 Runtime 没装**:Win11 自带,Win10 1809 以下要手动装,见 https://developer.microsoft.com/microsoft-edge/webview2/
- **显卡驱动问题**:更新显卡驱动
- **日志看哪里**:`%APPDATA%\nai-studio-desktop\logs\`

### Q2: 出图一直 500 / 失败
- **看错误提示**:常见是 NAI 5xx(服务端问题)、429(限流)、403(WAF 拦)
- **配代理**:Cloudflare WAF 用 TLS 指纹检测非浏览器,直连容易被拦。开 Clash / v2rayN 走 7890 端口
- **换 Key**:多配几个 API key,系统自动轮换

### Q3: Upscale 2x/4x/8x 区别
- **2x** ≈ 2-3 秒,质量够用,文件最小
- **4x** ≈ 7-8 秒,质量高
- **8x** ≈ 80+ 秒(4x AI + 2x 二次采样),质量最高但吃内存

第一次 upscale 会自动下载 Real-ESRGAN binary + 模型(约 50MB),等几十秒。

### Q4: 出图速度慢
- 832x1216 竖图:3-15 秒(看 NAI 服务器负载)
- 1216x832 横图:3-15 秒
- 2048x2048 大图:30-60 秒,可能触发 NAI 队列

### Q5: 数据丢了 / DB 损坏
- 删 `%APPDATA%\nai-studio-desktop\nai-studio.db*` 三个文件
- 重新启动 EXE,会自动创建空 DB
- 从备份恢复:把 `nai-studio.db` 覆盖回去

### Q6: 能装在 U 盘上带走吗
可以。EXE 拷到 U 盘,首次启动时 `%APPDATA%` 还在本机,但你可以:
1. 改用便携版(导出 `nai-studio-desktop` 文件夹)
2. 走 mklink 软链接把 APPDATA 目录指到 U 盘

### Q7: 跟旧 PHP 版能共存吗
可以,数据完全兼容:
- 旧版:`D:\anima\nai-studio\user-data\`
- 新版:`%APPDATA%\nai-studio-desktop\`
- 把旧版 `user-data\storage\*` + `nai-studio.db` 复制到新版对应位置即可

### Q8: 旧 PHP 版能不能继续用
可以,完整保留在 `D:\anima\nai-studio-archive\`。但建议切到新版(更快、更稳、占空间小)。

### Q9: 怎么完全卸载
1. 删 EXE
2. 删 `%APPDATA%\nai-studio-desktop\`

没了,不写注册表,不污染系统。

## 🆘 8. 报告问题

提 issue 时附上:
1. **EXE 版本**(`%APPDATA%\nai-studio-desktop\VERSION` 文件)
2. **最近日志**(`%APPDATA%\nai-studio-desktop\logs\` 里最新的 `.log`)
3. **操作步骤** — 怎么触发的
4. **错误截图** — 出错时的窗口
